# push_swap

Sorting algorithm project from the 42 School curriculum. The goal is to sort a list of integers using only two stacks (`a` and `b`) and a limited set of operations, generating the shortest possible instruction sequence.

The program implements four sorting strategies of different complexity and automatically selects the most appropriate one based on the input's degree of disorder.

### Checker (bonus)

The `checker` program verifies that the operations generated by `push_swap` correctly sort the stack:

```bash
ARG="4 67 3 87 23"
./push_swap $ARG | ./checker $ARG
# OK
```

- Prints `OK` if the stack is sorted and `b` is empty
- Prints `KO` if not sorted
- Prints `Error` on stderr for invalid arguments or unknown instructions

### Examples

```bash
# Basic sorting
./push_swap 5 3 1 4 2

# String input (quoted)
./push_swap "5 3 1 4 2"

# Force simple strategy and count operations
./push_swap --simple 5 4 3 2 1 | wc -l

# Verify with checker
ARG="4 67 3 87 23"
./push_swap --complex $ARG | ./checker $ARG

# Benchmark mode
./push_swap --bench 5 3 1 4 2 2>/dev/null

# Test with 500 random numbers
shuf -i 0-9999 -n 500 > args.txt
./push_swap $(cat args.txt) | wc -l
```

### Error Handling

The program prints `Error` on stderr and exits for:
- Non-integer arguments
- Arguments exceeding `int` range
- Duplicate values
- Invalid flags
- More than one strategy flag at a time

## Available Operations

| Operation | Description |
|---|---|
| `sa` | Swap the first two elements of `a` |
| `sb` | Swap the first two elements of `b` |
| `ss` | `sa` + `sb` simultaneously |
| `pa` | Push top of `b` to top of `a` |
| `pb` | Push top of `a` to top of `b` |
| `ra` | Rotate `a` upward (first becomes last) |
| `rb` | Rotate `b` upward |
| `rr` | `ra` + `rb` simultaneously |
| `rra` | Reverse rotate `a` (last becomes first) |
| `rrb` | Reverse rotate `b` |
| `rrr` | `rra` + `rrb` simultaneously |

---

## Algorithms

### 1. Simple вЂ” O(nВІ)

**Strategy:** Selection sort adapted for two stacks.

Each iteration finds the minimum in stack `a`, rotates it to the top (using `ra` or `rra` depending on shortest path), and pushes it to `b`. Once all elements are in `b`, they are pushed back to `a`. For sizes <= 5, optimized special cases are used (`sort_two`, `sort_three`, `sort_five`).

---

### 2. Medium вЂ” O(nв€љn)

**Strategy:** Chunk sort.

The value range is divided into в€љn uniform blocks (chunks) scaled by a 1.4 factor. Each pass pushes elements belonging to the current chunk to `b`, minimizing rotations. Once all chunks are in `b`, `a` is reconstructed by extracting maximums with `pa`.

---

### 3. Complex вЂ” O(n log n)

**Strategy:** Radix sort LSD (Least Significant Digit) with index representation.

Values are normalized to indices 0..n-1 for binary processing. For each bit (LSB to MSB), elements are distributed between `a` and `b` based on the bit value, then regrouped. Number of passes is logв‚‚(n).

---

### 4. Adaptive вЂ” automatic selection

**Strategy:** Combines the three algorithms above based on a disorder index computed before any operations.

The disorder index measures the fraction of pairs (i, j) with i < j where `a[i] > a[j]`:

```
disorder = number of inversions / total possible pairs Г— 100
```

| Disorder Range | Algorithm | Complexity |
|---|---|---|
| < 20% | Simple (nearly sorted) | O(nВІ) |
| 20% - 50% | Chunk sort | O(nв€љn) |
| >= 50% | Radix sort LSD | O(n log n) |
